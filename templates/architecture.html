设计理念

为什么用 多智能体？

单一模型难以在所有领域都表现出色。本架构将复杂任务拆分给专注单一领域的专家智能体，由一个调度器智能分发——每个 Agent 只做自己最擅长的事。

🧭

智能路由

Supervisor 分析每条消息，动态决定交由哪位专家处理，而非一个模型包揽一切。任务完成后确定性终止，避免无效循环。

🔌

状态图编排

基于 LangGraph 的 StateGraph 声明节点与条件边，路由逻辑可视化、可推理。消息通过 reducer 自动累积，节点只需产出增量。

💾

会话持久化

MemorySaver 检查点按 thread_id 隔离每个对话。服务重启后历史完整保留，支持随时回放与多轮上下文累积。

系统架构

星型拓扑调度模型

所有专家不直接通信，统一由 Supervisor 居中调度。每轮交互形成「路由 → 执行 → 回报 → 再评估」的闭环。

运行机制

一次对话的完整生命周期

从用户按下回车，到专家回复渲染到屏幕——每一步都可追踪、可调试。

1

消息注入

用户输入经 /api/chat 进入，包装为 HumanMessage 注入 ChatState.messages，携带唯一 thread_id。

2

Supervisor 路由决策

调度器节点读取完整对话历史，判断：若本轮已有专家回复 → 确定性返回 FINISH；否则调用 LLM 从 coder / researcher / writer 中选出最匹配者。

3

条件边分发

StateGraph 的 conditional_edges 根据 state["next"] 将控制权路由到对应专家节点，或将 FINISH 映射到图终点。

4

专家执行

被选中的专家以其专属 persona system prompt + 对话上下文调用 LLM，生成回复并追加为 AIMessage，随后把控制权交还 Supervisor。

5

闭环与持久化

Supervisor 检测到已有回复 → 返回 FINISH，本轮终止。MemorySaver 将最终状态检查点写入内存，整个 trace 流式返回前端实时渲染。

核心特性

工程化设计要点

🎯

确定性终止

不依赖 LLM 判断"是否完成"——用代码检测最后一条消息后是否已有 AI 回复，杜绝推理模型的误判与无限循环。

🛡️

双模降级

有 API Key 走真实 LLM，无 Key 自动降级到确定性 MockLLM。UI 实时显示当前模式，演示零门槛。

🌐

中英双语

前端 i18n 字典驱动，一键切换中英文界面，选择记忆于 localStorage，含中文输入法 IME 安全的回车发送。

📊

路由可观测

流式返回执行 trace，前端实时渲染 Supervisor 的路由决策链与各专家回复，每次跳转清晰可见。

🔌

即插即用

新增专家只需在 build_team() 加一行 persona，图的节点、边、路由自动接线，零样板代码。

⚡

Checkpointer 持久化

MemorySaver 按 thread 隔离存储，重启不丢历史。可平滑替换为 SQLite / Postgres Saver 实现跨进程。

# StateGraph: 声明节点 + 条件路由边
graph = StateGraph(ChatState)

graph.add_node("supervisor", supervisor_node)
for name in team:
    graph.add_node(name, make_specialist(name))

graph.set_entry_point("supervisor")

# Supervisor 根据决策路由到专家或终点
graph.add_conditional_edges(
    "supervisor",
    lambda state: state["next"],
    {n: n for n in team} | {"FINISH": END},
)

# 每个专家执行后回到 Supervisor 再评估
for name in team:
    graph.add_edge(name, "supervisor")

app_graph = graph.compile(checkpointer=memory)
# 流经所有节点的共享状态
class ChatState(TypedDict):
    # add_messages reducer: 节点只返回新消息，自动累积
    messages: Annotated[List[BaseMessage], add_messages]
    next: str            # Supervisor 的路由决策
    active_agent: Optional[str]  # 当前发言的专家
    turn_count: int      # 跳数安全边界 (MAX_TURNS=6)

# Supervisor 的确定性终止逻辑
def supervisor(state):
    msgs = state["messages"]
    last_human = find_last_human(msgs)
    has_reply = any(m.type=="ai" and i > last_human
                   for i, m in enumerate(msgs))
    if has_reply:
        return {"next": "FINISH"}  # 代码判定，不靠 LLM
    return {"next": llm_pick_specialist(msgs)}
# FastAPI: 流式执行 + 检查点持久化
@app.post("/api/chat")
async def chat(req: ChatRequest):
    thread_id = req.thread_id or uuid()
    config = {"configurable": {"thread_id": thread_id}}

    inputs = {"messages": [HumanMessage(req.message)]}
    # 流式执行，收集每个节点的增量
    stream = app_graph.stream(inputs, config, stream_mode="updates")
    trace = extract_trace(stream)

    # 从持久化状态取最终回复
    snap = app_graph.get_state(config)
    final = snap.values["messages"][-1].content

    return {"thread_id": thread_id, "trace": trace,
            "final": final, "llm_mode": mode}
